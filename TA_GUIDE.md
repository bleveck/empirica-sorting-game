# TA Guide — POLI 172 Group Sorting Game

## The Basics

Students play a group-sorting game on their phones or laptops by visiting **poli172.games** in a browser. They register with their name, play up to 20 rounds, and fill out a short reflection survey.

There are **two sessions** in the same class:

1. **Session A** — switching groups is free
2. **Session B** — each switch costs 1 point

Between sessions, students need to refresh the page and re-enter the same name/email.

## Helping Students Connect

- The URL is **poli172.games** (any browser — Chrome, Safari, etc.)
- Must be **https** — if a student types `http://poli172.games` it should redirect, but if it doesn't, tell them to type `https://poli172.games`
- Works on phones, tablets, and laptops — no app needed
- If the page won't load, ask if they're on campus Wi-Fi

## Registration

- Students enter their **first/last name** and **UC Merced email prefix**, then click Join
- "Game not available" usually means the batch hasn't been started yet — wait a moment and try again
- After registering, they'll see the rules, then click "I understand. Start the game."
- They'll wait in a lobby until everyone joins; then the game starts for everyone simultaneously

## During the Game

- Each player is randomly assigned a **type** — `A` (blue) or `B` (orange) — that stays the same the whole game
- Each player starts in either **Group 1** or **Group 2**, randomly
- Each round has two screens:
  1. **Result** — shows your type, your group, both groups' A/B counts, your fit/congestion payoff, and your running score. Click **Continue** when ready (or wait for the timer).
  2. **Switch decision** — choose **Stay** or **Switch**. In Session B the Switch button shows the cost.
- The game ends after 20 rounds *or* sooner if the class perfectly sorts (all A's in one group, all B's in the other).

## Common Issues

| Problem | What to Do |
|---------|------------|
| Screen blank / stuck on loading | Refresh the page |
| "Please wait for other players" stays | Normal during lobbies and between stages — it clears when everyone is ready |
| Closed the browser | Reopen and go to poli172.games — it should reconnect automatically |
| "No games available" | The batch hasn't started yet, or it ended — check with the instructor |
| Student arrived late | They can still join if the lobby is still open. If the batch has started, they wait for the next one |
| Score went negative | Normal — congestion costs can outweigh fit. The game allows negatives |
| Confused about their type | Type is shown in the top-left header and on every round screen — `A` is blue, `B` is orange |
| Confused about "switch" | "Switch" moves you to the other group **starting the next round**. In Session B it costs 1 point |

## What NOT to Worry About

- You don't need to touch the server or admin panel
- You don't need to know the game theory — just help students get connected and playing
- Between Session A and Session B, students re-enter the same info to keep their data linked
- If something seems broken for everyone (not just one student), flag it to the instructor
